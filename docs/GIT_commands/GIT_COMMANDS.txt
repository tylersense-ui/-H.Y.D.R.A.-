# 🐍 H.Y.D.R.A. v0.1.0-GENESIS - Commandes Git

## ÉTAPE 1: Initialisation du Repository Local

```bash
cd /mnt/user-data/outputs/HYDRA

# Initialiser Git
git init

# Configurer les informations de commit (si nécessaire)
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

## ÉTAPE 3: Configuration du Remote GitHub

```bash
# Définir la branche principale
git branch -M main

# Ajouter le remote GitHub
git remote add origin https://github.com/tylersense-ui/-H.Y.D.R.A.-.git

# Vérifier le remote
git remote -v
```

## ÉTAPE 4: Premier Push

```bash
# Pousser sur GitHub
git push -u origin main
```

## ÉTAPE 5: Vérification

```bash
# Vérifier que tout est bien poussé
git log --oneline
git status
```

Ensuite, visitez: https://github.com/tylersense-ui/-H.Y.D.R.A.-

## CONVENTIONS DE COMMIT

Utiliser les préfixes suivants:

- `feat:` - Nouvelle fonctionnalité
- `fix:` - Correction de bug
- `docs:` - Documentation uniquement
- `refactor:` - Refactoring sans changement de fonctionnalité
- `perf:` - Amélioration de performance
- `test:` - Ajout/modification de tests
- `chore:` - Tâches de maintenance

Exemples:
```bash
git commit -m "feat: add FFD RAM packing algorithm"
git commit -m "fix: telemetry daemon crash on empty server list"
git commit -m "docs: update README with Phase 2 roadmap"
```

## EN CAS DE PROBLÈME

### Remote déjà existant

```bash
git remote remove origin
git remote add origin https://github.com/tylersense-ui/-H.Y.D.R.A.-.git
```

### Push rejeté (force push - ATTENTION!)

```bash
# SEULEMENT si vous êtes sûr de vouloir écraser le remote
git push -f origin main
```

### Vérifier l'état du repo

```bash
git status
git log --oneline --graph
git remote -v
```
